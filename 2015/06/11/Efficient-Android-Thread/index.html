Efficient Android Thread

主要介绍了android里使用Thread优化UI更新（optimize UI on Thread）及实现异步（Asynchronous）的技巧。

实现异步的方法

  • Thread
  • Executor
  • HandlerThread
  • AsyncTask
  • IntentService
  • AsyncQueryHandler
  • Loader

Thread

直接使用java线程实现异步操作。
注意：UI更新必须使用主线程否则会报异常。

易错点；

  • 线程无法保留（non-retained）
  • 无法取消
  • 每次使用都需要重新启动

适用情况：
只需要执行一次的操作

Executor

任务执行框架



线程池

  • Executor管理使用线程池工作队列来管理线程
  • 减少了创建线程的资源消耗
    线程池的几种状态

    易错点：
    任务从顺序执行转换为同步执行的时候会失去线程安全性
    使用情况：
  1. 同步执行任务
    • 多次网络请求
    • 图像处理
    • 获取用例
  2. 管理生命周期，检测任务执行
  3. 使用多平台

HandlerThread

  • 封装了Looper对象的Thread
  • 轮询（Loop）MessageQueue对象处理信息
  • 处理Message，Runable实现的操作

与handler配合使用

适用情况:

  • 保持线程长期存在
  • 顺序处理信息
    • 避免多次按键导致的任务多次执行
    • 状态机
    • 控制信息处理的细节

AsyncTask

  • 封装了Handler与Looper
  • 使用的Executor框架
  • UI更新与后台操作回调

易错点：

  • asyncTask处理应用全局操作
  • execute，cancelation，create，start




IntentService

  • 有工作队列的service
  • 独立线程处理Intents

适用情况:

  • 连续执行从其他组件内解耦的任务
  • 与UI线程无关
  • REST客户端(使用ResultReceiver作为回调)

AsyncQueryHandler

  • 兼容API1
  • 异步操作ContentResolver
  • 包装了HandlerThread

缺点：

  • 无法管理游标
  • 没有内容监测
  • 配置更改后数据不能保留
  • 后台线程不能强行终止

Loader

  • 有可用的兼容包
  • 后台线程加载数据
  • 可监测数据变化
  • 配置更改后数据保留
  • 与Activity和Fragment的生命周期关联

易错点:

  • activity或fragment被破坏后Loader停止加载数据
